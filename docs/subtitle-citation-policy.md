# 자막 · 인용 · 출처 정책

기준일: 2026-07-21  
상태: **초안 (Draft)** — 구현·배포 전 체크리스트로 사용  
관련: `docs/culture-lexicon-learning-spec.md`, `scripts/build-klassic-quotes.mjs`, `scripts/lib/content-rating.mjs`

---

## 1. 목적

K-culture 학습 콘텐츠(명대사, 드라마·영화 맥락, 문화고유어)를 다루면서도:

1. **저작권·약관을 침해하지 않고**
2. **학습자에게 안전한 수위**를 유지하며
3. **출처를 투명하게** 남기기 위한 규칙이다.

이 문서는 “무엇을 공개 데이터·GitHub Pages에 실을 수 있는가”의 **허용 경계**를 정의한다.

## 2. 원칙 (한 줄)

| # | 원칙 |
|---|------|
| P1 | **원문 대본·공식/비공식 자막 파일을 수집·재배포하지 않는다.** |
| P2 | 공개 학습 문장은 **교육용 시드·패러프레이즈·공개 API·라이선스 허용 자료**만 사용한다. |
| P3 | 작품 **제목·장르·문화 현상**은 맥락으로 쓸 수 있으나, **대사 전문 복제**는 하지 않는다. |
| P4 | 학술 논문은 **설계·전략 참고**이며, 논문 속 예문 대사를 **대량 시드로 옮기지 않는다.** |
| P5 | 수위(비속어·폭력)는 **기본 clean**, 학습자가 필터로 확장한다. |
| P6 | 모든 배포 데이터 묶음에 **출처 유형**을 메타로 남긴다. |

## 3. 허용 / 금지

### 3.1 허용 (공개 repo · Pages 배포 OK)

| 유형 | 예시 | 조건 |
|------|------|------|
| 교육용 시드 | “오징어 게임은 넷플릭스에서 유명해진 한국 시리즈예요.” | 원문 대사 아님, 짧은 설명·회화 연습 |
| 문화고유어 설명 | 깐부, 달고나, 무궁화 꽃이 피었습니다 (뜻·상황·연습 문장) | 사전·교재·자작 설명 수준 |
| 공개 API 인용 | klassic-quote-api 응답을 빌드 스냅샷 | API 약관·이용 조건 준수, 출처 표기 |
| 공공데이터 | 관광공사 TourAPI, 세종 어휘 CSV | 해당 이용약관·출처 표기 |
| 차트 메타 | Melon 순위·곡명·아티스트 + **자작** 학습 표현 | 가사 전문 미포함 |
| 학술 참고 | KCI 논문의 **전략 프레임**(언어유희·비속어·맥락·문화고유어) | 설계 문서·스펙에 인용; 예문 대량 복제 금지 |
| 짧은 교육 인용 | 이미 널리 알려진 한 줄 수준의 인용 | 출처·작품명 표기, 분량 최소화, 상업 전문 배포 금지 |

### 3.2 금지 (구현·커밋·배포 모두)

| 유형 | 예시 | 이유 |
|------|------|------|
| 비공식 SRT/ASS 등 | YTS, 블로그 “자막 다운로드”, 리핑 파일 | 무단 배포물 취급 소지 |
| 스트리밍 자막 추출 | 넷플릭스 등 클라이언트/네트워크에서 자막 덤프 | 약관·저작권 위반 소지 |
| 대본 전문·대량 대사 DB | 작품 에피소드 단위 대사 목록을 seed로 적재 | 저작물 재배포 |
| 논문 예문 일괄 이전 | 학술지 ST/TT 대조표를 학습 카드로 통째 복사 | 저작·인용 범위 초과 위험 |
| 가사 전문 | K-pop 곡 전체 가사 | 저작권 |
| 수위 무필터 기본 노출 | 욕설·잔인 묘사 기본 ON | 학습 안전·브랜드 |

### 3.3 회색 지대 (기본 금지, 별도 결정 필요)

| 유형 | 처리 |
|------|------|
| 사용자 기기 로컬 SRT | **서버 업로드·repo 커밋 없이** 브라우저 메모리에서만 처리하는 기능은 검토 가능. UI에 “합법 보유·자작 자료만” 경고 필수. 샘플로 상업 영화 SRT를 제공하지 않음. |
| 공정이용·교육 인용 | 법무/정책 검토 전 **한 줄 시드 + 출처** 수준을 넘지 않음. |
| CC/연구 코퍼스 | 라이선스 전문 확인 후 `source.license` 필드에 기록된 경우만 허용. |

## 4. 출처 유형 코드 (`source` / `sourceType`)

빌드 산출물·시드에 가능한 한 아래 값을 사용한다.

| 코드 | 의미 | 공개 배포 |
|------|------|-----------|
| `seed` | 팀이 작성한 교육용 문장 | ✅ |
| `seed-culture` | 문화고유어·현상 설명 시드 | ✅ |
| `api:klassic-quote` | klassic-quote-api | ✅ (약관 준수) |
| `public-data:kto` | 한국관광공사 등 공공데이터 | ✅ |
| `chart:melon-meta` | 차트 메타 + 자작 미션 | ✅ |
| `user-local` | 사용자 로컬 파일 (브라우저 only) | ❌ repo/Pages 저장 금지 |
| `unofficial-sub` | 비공식 자막 | ❌ 사용 금지 |
| `paper-excerpt` | 논문 예문 직접 전재 | ❌ 기본 금지 |

기존 `klassic-quotes`의 `source: "seed"` 관례를 유지하고, 문화어 확장 시 `seed-culture`를 권장한다.

## 5. 수위(content rating) 정책

기존 파이프라인: `scripts/lib/content-rating.mjs` → `clean | mild | mature`.

| 등급 | 포함 | UI 기본 |
|------|------|---------|
| `clean` | 일상·문화 설명, 비폭력 회화 | **기본 선택** |
| `mild` | 거친 말투 암시, 가벼운 비난 표현 | 옵트인 |
| `mature` | 노골적 비속어·폭력 맥락 | 옵트인, 경고 |

- 드라마·영화 맥락 카드라도 **기본 문장은 clean**.
- “작품에는 비속어가 많다”는 **메타 설명**으로 두고, 원문 욕설을 기본 카드에 넣지 않는다.
- 명대사 필터 UI(`quote-rating-filter`)와 동일한 축을 문화어·시드에도 적용한다.

## 7. 학술 자료 인용 규칙

예: KCI `FI002973700` — 신상범·이재성 (2023). 드라마 「오징어 게임」의 자막번역 전략 연구. *통번역교육연구* 21(1).

| 할 수 있음 | 하지 않음 |
|------------|-----------|
| 논문의 **분석 축**(언어유희·비속어·맥락·문화고유어)을 스펙에 반영 | ST/TT 대조 예문을 학습 DB에 일괄 입력 |
| 자막 **형식 제약**(줄 길이·시간)을 UX 가이드로 인용 | “넷플릭스 공식 가이드라인 전문”을 재배포 |
| 참고문헌 목록에 정식 서지 표기 | 논문 PDF를 저장소에 커밋 (라이선스·배포 권한 불명 시) |

로컬 작업용 추출 마크다운(`_workspace/`)은 **Git 제외 권장**. 공유가 필요하면 서지 정보와 링크만 문서에 남긴다.

## 8. 로컬 SRT 기능 (향후) 게이트

구현 시 필수 조건:

1. **클라이언트 only** — 파일 내용 서버 전송·analytics 원문 전송 없음  
2. **repo/cache에 사용자 SRT 저장 금지**  
3. 첫 사용 시 동의 문구:  
   > 본인이 합법적으로 보유·작성한 자막/대본만 불러오세요. 스트리밍 서비스 자막이나 무단 배포 파일을 사용하지 마세요.  
4. 샘플 파일로 상업 영화 SRT를 제공하지 않음  
5. 매칭 결과는 세션 메모리 또는 **사용자 명시 동의 하의** localStorage 요약만 (원문 전체 저장 지양)

이 게이트를 통과하기 전에는 로컬 SRT UI를 제품 기본 경로에 넣지 않는다.
