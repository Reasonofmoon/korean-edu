# 첫 학습 세션 플로우

기준일: 2026-05-13 KST

## 목적

app-factory의 `learning-content-app` / `builder-mini-behavior` 관점에서 첫 세션의 행동 루프를 명확히 한다.

첫 세션의 성공은 사용자가 전체 데이터를 탐색하는 것이 아니라, 어휘 하나를 골라 `뜻 확인 -> 듣기/읽기 -> 핵심 표현 확인 -> 퀴즈 -> 완료/복습`까지 진행하는 것이다.

## Flow Map

1. 진입
   - 사용자는 오늘의 추천 카드에서 시작한다.
   - 시스템은 아직 완료하지 않은 첫 학습 활동을 추천한다.

2. 어휘 선택
   - 사용자는 추천 CTA 또는 어휘 카드에서 어휘를 선택한다.
   - 선택된 어휘가 세션 플로우의 기준이 된다.

3. 맥락 확인
   - 뜻, 단원, 예문, 현장 미션을 본다.
   - 이 단계는 사용자의 이해 부담을 줄이기 위해 기본 완료로 간주한다.

4. 듣기/읽기
   - 오디오가 있으면 듣기 완료 이벤트를 저장한다.
   - 오디오가 없으면 장소 설명/대본 기반 읽기 카드와 `읽기 완료했어요` 버튼으로 대체한다.

5. 핵심 표현 확인
   - 사용자가 `표현 확인했어요` 버튼을 누르면 expression checked 상태를 저장한다.

6. 퀴즈
   - 생성된 퀴즈를 풀고 정답/오답 상태를 저장한다.
   - 오답 횟수는 복습 우선순위에 사용한다.

7. 완료/복습
   - 완료 조건은 `퀴즈 전체 정답 + 필요한 오디오 완료 + 핵심 표현 확인`이다.
   - 완료된 어휘는 배지로 표시되고, 미완료/오답 퀴즈는 복습 카드에 남는다.

## State Machine

| State | 진입 조건 | 주요 UI | 다음 전이 |
| --- | --- | --- | --- |
| `pick` | 세션 시작, 선택 어휘 없음 | 어휘 카드, 추천 카드 | 어휘 선택 후 `context` |
| `context` | 어휘 선택됨 | 뜻, 예문, 현장 미션 | 자동으로 `listen` 또는 `expressions` |
| `listen` | 오디오 또는 읽기 대체 콘텐츠가 아직 완료 안 됨 | 듣기 카드, 오디오 플레이어, 읽기 완료 카드 | audio ended 또는 읽기 완료 클릭 후 `expressions` |
| `expressions` | 핵심 표현이 아직 확인되지 않음 | 핵심 표현 목록, 확인 버튼 | 확인 버튼 클릭 후 `quiz` |
| `quiz` | 미풀이/오답 퀴즈가 있음 | 퀴즈 카드 | 전체 정답 후 `complete` |
| `complete` | 완료 조건 충족 | 완료 배지, 복습 카드 | 다음 추천 어휘로 이동 |

## UI 반영

- `#session-flow`를 추가해 현재 선택 어휘의 단계 흐름을 시각화했다.
- 단계 버튼을 누르면 해당 섹션으로 이동한다.
- 선택 어휘 상세 상단 CTA가 현재 상태에 따라 바뀐다.
- 오디오가 없는 어휘는 읽기 완료 상태를 `localStorage`에 저장한다.
- 핵심 표현 확인 상태를 `localStorage`에 저장한다.
- 완료 조건에 핵심 표현 확인을 포함했다.
- 완료된 어휘에는 다음 미완료 어휘로 이동하는 CTA를 표시한다.

## 다음 개선

- `context` 단계를 실제 확인 버튼으로 바꿀지 검토한다.
- 읽기 대체 콘텐츠의 품질을 대표 장소 설명/문화 이야기/예문 우선순위로 더 정교화한다.
- 완료 후 다음 추천 어휘의 선정 기준을 관심사, 카테고리, 오답 이력 기반으로 개선한다.
